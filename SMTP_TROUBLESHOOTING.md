# Решение проблемы с отправкой Email

## Проблема
DNS не может разрешить домен `mail.alliance-pravo.ru` - ошибка `getaddrinfo EAI_AGAIN`.

## Возможные решения

### 1. Проверьте правильный адрес SMTP сервера

В панели управления хостингом (ISPmanager) проверьте:
- Правильный адрес SMTP сервера
- Может быть `smtp.alliance-pravo.ru` вместо `mail.alliance-pravo.ru`
- Или IP-адрес сервера напрямую

### 2. Используйте IP-адрес вместо домена

Если знаете IP-адрес почтового сервера, можно использовать его напрямую в `.env`:
```
VITE_SMTP_HOST=XXX.XXX.XXX.XXX
```

### 3. Проверьте настройки в ISPmanager

В панели управления почтой должны быть указаны:
- **Имя сервера входящей почты:** обычно `mail.alliance-pravo.ru` или `smtp.alliance-pravo.ru`
- **Имя сервера исходящей почты:** обычно то же самое

### 4. Попробуйте альтернативные варианты

В файле `.env` попробуйте изменить:
```
VITE_SMTP_HOST=smtp.alliance-pravo.ru
```
или
```
VITE_SMTP_HOST=mail.alliance-pravo.ru
```

### 5. Проверьте доступность сервера

Если сервер доступен только из внутренней сети хостинга, то:
- На локальной машине отправка не будет работать
- Нужно использовать VPN или деплоить на сервер хостинга

## Что нужно сделать

1. **Откройте панель ISPmanager** и проверьте настройки почты для `info@alliance-pravo.ru`
2. **Найдите правильный адрес SMTP сервера** (может быть указан в настройках)
3. **Обновите `.env` файл** с правильным адресом
4. **Перезапустите сервер** и протестируйте снова

## Альтернативное решение

Если SMTP сервер недоступен с локальной машины, можно:
- Использовать внешний SMTP сервис (Gmail, Yandex, Mail.ru)
- Или деплоить на Vercel, где serverless функция будет работать

## Для тестирования локально

Можно временно использовать тестовый SMTP сервер или настроить локальный почтовый сервер для разработки.
